The four models at a glance

The whole lecture is these four shapes. Learn to recognise the shape, and you know which one to fit.

ModelStata syntaxShapeClinical use
Exponential decaynl (y={a}*exp({b}*x)), b<0Falls fast, then flattensDrug clearance, pain resolution
Exponential growthnl (y={a}*exp({b}*x)), b>0Rises ever fasterViral load (e.g. early Ebola), tumour growth
Michaelis-Mentennl (y={vmax}*x/({km}+x))Rises then saturatesDose-response, receptor binding
Logistic (sigmoid)nl (y={asym}/(1+exp({rate}*(x-{inflect}))))S-curve: lag, surge, plateauGrowth curves, epidemic case counts

The rhythm: plot → guess → fit → check

Whether it's a drug clearing from blood or an outbreak's case count climbing, biological change bends, and non-linear models aren't solved by a formula, they're searched for. Your job is to start the search well and check where it landed.

⌨️ Stata command 
* 1. LOOK   plot the raw data first
* 2. GUESS  read starting values off the plot
* 3. FIT    nl (y = ...), initial(...)
* 4. CHECK  did it converge? are the residuals clean?
💡 What this means
Linear regression lets you skip to the fit. Non-linear regression does not: Stata improves your starting guess by trial-and-error until it converges. A bad guess can wander off forever, so we always plot first, guess from the plot, fit, then check.

Model 1: Exponential decay (real auto data)

🎯 What this is & why it matters
The model: a curve that falls fast, then flattens toward a floor it never crosses. Why it matters: it is exactly how a drug clears from the blood, so a single rate gives you the half-life a clinician doses by.
🍵
🧩 Everyday version
Picture a hot cup of tea left on the table. It cools fast while it's piping hot, then slower and slower as it nears room temperature, and it never gets colder than the room. That "quick at first, then flattening toward a floor" shape is exponential decay. A drug clearing from the blood, a fading bruise, a phone battery draining, all the same curve. Today the stand-in is a car's fuel economy falling as the car gets heavier.

Dataset: the public auto-mpg.csv (398 cars). MPG falls steeply for light cars then flattens, the same shape as a drug concentration falling as body mass rises. (Your do-file uses Stata's 74-car sysuse auto: same shape, smaller-magnitude coefficients.)

⌨️ Stata command 
import delimited auto-mpg.csv, clear   // or: sysuse auto, clear
describe
list mpg weight in 1/5
💡 What this means
mpg is the outcome (a drug-concentration stand-in); weight the predictor (body mass). Load, then eyeball a few rows before modelling.
⌨️ Stata command 
twoway scatter mpg weight, ///
    title("MPG vs Vehicle Weight") xtitle("Weight (lbs)") ytitle("MPG")
📈 Graph, generated from the data
💡 What this means
Model: MPG = a · e^(b·Weight). The curved, decaying cloud is the evidence that a straight line is wrong. a = MPG at zero weight (the ceiling); b = decay rate (negative ⇒ MPG falls as weight rises).
⌨️ Stata command 
* starting values are a rough GUESS off the plot, the search refines them
nl (mpg = {a} * exp({b} * weight)), initial(a 40 b -0.0005)
📋 Output, real least-squares fit (computed in Python)
mpg      Coef.  Std.err.       t  P>|t|  [95% conf.  interval]
---  ---------  --------  ------  -----  ----------  ---------
/a      66.063     2.248   29.39  0.000      61.644     70.482
/b   -0.000363  0.000013  -28.09  0.000   -0.000388  -0.000337
📖 Read the output
First time reading an nl table, take it column by column. Each row is one parameter the search found. Coef. is the estimate: a ≈ 66.1 (the ceiling MPG) and b ≈ -0.00036 (negative ⇒ decay). Std. err. is how much that estimate would wobble in a fresh sample of cars. t is Coef.÷Std.err., and P>|t| turns that into the chance of seeing this if the true value were zero, both read 0.000, so neither is a fluke. The [95% conf. interval] is the plausible range for the true value, and neither one crosses 0. So: read just two numbers, a and b, and you've read the whole curve.
💡 What this means
This is a real least-squares fit. a ≈ 66.1 (ceiling MPG) and b ≈ -0.00036 (negative ⇒ decay). Both p < 0.001, so neither is plausibly zero. Add vce(robust) for robust SEs when the residual spread isn't even, the coefficients don't move, only the error bars do.
🩺 Clinical translation
ParameterEstimate95% CIClinical meaning
a66.1 MPG(61.6, 70.5)Maximum MPG at zero weight (the ceiling)
b-0.00036(-0.00039, -0.00034)Each +1000 lb lowers MPG by ~30%
⚠️ Watch out
If Stata prints “convergence not achieved”, the numbers are meaningless, fix initial() and refit.
⌨️ Stata command 
predict fitted_exp, yhat
predict resid_exp, resid
rvfplot, yline(0)
qnorm resid_exp
histogram resid_exp, normal
📈 Graph, generated from the data
💡 What this means
The standard health-check, computed from the real residuals: a patternless cloud around zero (left) and points hugging the diagonal (right) both say the model fits well. A funnel or U-shape would mean the model shape is wrong.
⌨️ Stata command 
nlcom (half_life: ln(2) / -_b[/b])
📋 Output, real least-squares fit (computed in Python)
       half_life |  Coef. = 1,910.52   [95% conf. int.: 1,785.55, 2,054.29]
📖 Read the output
One line, one number: 1911 lbs, with its 95% CI in brackets. That's the weight gain that halves MPG. Stata carried the uncertainty in b through into a correct interval for you, no by-hand error propagation.
💡 What this means
nlcom turns parameters into a clinically intuitive quantity with a correct CI. Half-life = ln(2)/(−b) ≈ 1911 lbs: every extra ~1911 lb halves the MPG, exactly like a drug half-life.

Model 2: Logistic growth (an S-curve)

🎯 What this is & why it matters
The model: the S-curve, slow then fast then a ceiling. Why it matters: it is the shape of an outbreak's running case count and of any growth that runs into a limit, so it pins down both the ceiling and the tipping-point.
🗣️
🧩 Everyday version
Picture a juicy rumour spreading through a village. At first only one or two people know, so it crawls. Then everyone who hears it tells two friends and it explodes. Finally it slows again, because almost everyone has already heard it. Slow → fast → plateau: that S-shape is logistic growth. The very same curve describes life expectancy rising toward a ceiling, a crop growing, or an outbreak's running case count.

Dataset: life expectancy, 1900–2000. The S-shape, slow, then fast, then a ceiling, is the fingerprint of logistic growth (and, as we'll see, of an epidemic curve).

⌨️ Stata command 
sysuse uslifeexp, clear
list year le in 1/10
💡 What this means
year is the predictor, le (life expectancy) the outcome. The values rise and then level off.
⌨️ Stata command 
* Model:  le = asym / (1 + exp(rate*(year - inflect)))
twoway scatter le year
nl (le = {asym}/(1+exp({rate}*(year-{inflect})))), ///
    initial(asym 80 rate -0.1 inflect 1950)
estimates store logistic_model
📋 Output, real least-squares fit (computed in Python)
le           Coef.  Std.err.        t  P>|t|  [95% conf.  interval]
--------  --------  --------  -------  -----  ----------  ---------
/asym       78.325    0.2693   290.89  0.000      77.791     78.859
/rate      -0.0348  0.000767   -45.41  0.000     -0.0363    -0.0333
/inflect  1,890.35    0.4042  4676.72  0.000    1,889.55   1,891.15
📈 Graph, generated from the data
📖 Read the output
Read it the same way, now with three rows. asym ≈ 78.3 = the ceiling the curve climbs toward; inflect ≈ 1890 = the year of steepest change; rate = how sharp the bend is. Check the P>|t| and CI columns exactly as before, all three are firmly pinned down, so the S-curve is a real description of the data, not wishful drawing.
💡 What this means
Three meaningful parameters, really fitted: asym ≈ 78.3 is the ceiling (max life expectancy the curve approaches); inflect ≈ 1890 is the year of fastest growth; rate sets the steepness. Here the inflection sits just before our window, so 1900–2000 is the decelerating upper arm of the S, gains slowing as we near the ceiling.
⌨️ Stata command 
predict fitted_log, yhat
local asym=_b[/asym]
local rate=_b[/rate]
local inflect=_b[/inflect]
display "Predicted LE 2020: " `asym'/(1+exp(`rate'*(2020-`inflect')))
📋 Output, real least-squares fit (computed in Python)
Predicted LE 2020: 77.48
📖 Read the output
This line isn't a coefficient, it's a prediction. We fed the fitted curve a year it never saw (2020) and it returned 77.5 years, sitting just under the ceiling because the S has all but flattened. That is what a model is for: answering beyond the data you measured.
💡 What this means
Plug the fitted parameters into the formula for 2020 to forecast ≈ 77.5 years, close to the ceiling, because the curve has nearly flattened.
🦠 Outbreak lens
The same S-curve describes an outbreak's cumulative case count: a slow lag, an explosive surge, then a plateau as control measures bite and susceptibles run out. The logistic is the natural model for an epidemic curve.
⚠️ Watch out
Forecasting far past your data assumes the same curve keeps holding, be cautious.

Model 3: Michaelis–Menten (saturating dose–response)

🎯 What this is & why it matters
The model: response climbs as you raise the dose, then saturates at a maximum it cannot beat. Why it matters: it hands you the two numbers a pharmacologist lives by, the most a drug can do (Vmax) and the dose that gets you halfway there (the EC50, its potency).
🧽
🧩 Everyday version
Picture a dry sponge dropped in water. At first it drinks greedily; as it fills, each second adds less; once it is soaked it holds no more, however long you leave it in. The response rises fast, then levels off at a ceiling. The amount of water that gets the sponge halfway full is the same idea as a drug's EC50, the dose that delivers half of its biggest possible effect.

Simulated dose–response, so we know the truth (true Vmax=90, Km=25) and can grade the fit. Clinical analogy: receptor binding, response rises then saturates.

⌨️ Stata command 
clear
set seed 12345
set obs 40
gen dose = (_n-1)*(100/39)
gen response = (90*dose)/(25+dose) + rnormal(0,4)
💡 What this means
We simulate 40 doses (0→100 mg/kg) from a known curve plus noise, so the fit below can be graded against the truth (Vmax=90, Km=25).
⌨️ Stata command 
nl (response = {vmax}*dose/({km}+dose)), initial(vmax 90 km 25)
📋 Output, real least-squares fit (computed in Python)
response   Coef.  Std.err.      t  P>|t|  [95% conf.  interval]
--------  ------  --------  -----  -----  ----------  ---------
/vmax     89.175     3.330  26.78  0.000      82.433     95.916
/km       23.655     2.784   8.50  0.000      18.019     29.290
📈 Graph, generated from the data
📖 Read the output
Two rows. Vmax ≈ 89.2 = the saturating maximum response; Km ≈ 23.7 = the dose that gets you to half of that maximum (the EC50). Here's the honest test: because this data is simulated we know the truth was 90 and 25, and the fit lands on both. That agreement is your proof the method recovers reality, not just any curve.
💡 What this means
Vmax ≈ 89.2 is the saturating maximum response; Km ≈ 23.7 is the EC50, the dose giving half of Vmax. Both land on the true values (90, 25), so the model works.
🩺 Clinical translation
ParameterEstimate95% CIClinical meaning
Vmax89.2%(82.4, 95.9)Maximum achievable response
Km (EC50)23.7 mg/kg(18.0, 29.3)Dose for 50% of maximum effect
⌨️ Stata command 
predict fitted_mm, yhat
predict resid_mm, resid
rvfplot, yline(0)
predict cooksd_mm, cooksd
scatter cooksd_mm _n, yline(4/40)
💡 What this means
After fitting, overlay the fitted curve, check residuals-vs-fitted for pattern, and use Cook's distance (cut-off 4/n) to flag any single point that pulls the whole fit, a data-entry error or a genuinely unusual subject.

Model 4: Exponential growth (Ebola viral load & the Gulu 2000 outbreak)

🎯 What this is & why it matters
The model: exponential growth, doubling again and again. Why it matters: it is how an untreated viral load and an outbreak's early cases both climb, so the doubling time tells you how many days you have before the count explodes.
💰
🧩 Everyday version
Picture money in a savings account earning compound interest, or a single WhatsApp message forwarded on and on: one becomes two, two become four, four become eight. Early on nothing looks alarming, then it rockets. The number that matters is the doubling time: how long until it's twice as big. Untreated viral load in a patient, and an outbreak's early case count, both grow exactly this way, which is why a few days' delay costs so much.

Same exponential family as Model 1, but with a positive rate, so instead of decay we get explosive growth. We look at it at two scales: inside one patient, and across a population.

⌨️ Stata command 
clear
set seed 67890
set obs 20
gen day = _n
* Ebola viral load climbing in an untreated patient
gen viral_load = 100*exp(0.3*day) + rnormal(0,200)
nl (viral_load = {a}*exp({b}*day)), initial(a 100 b 0.3)
📋 Output, real least-squares fit (computed in Python)
viral_load   Coef.  Std.err.       t  P>|t|  [95% conf.  interval]
----------  ------  --------  ------  -----  ----------  ---------
/a          94.077     3.350   28.08  0.000      87.039    101.116
/b          0.3033  0.001887  160.71  0.000      0.2993     0.3072
📈 Graph, generated from the data
📖 Read the output
The same two-parameter exponential as Model 1, but read the sign of b. Here b ≈ 0.303 is positive: growth, not decay. a ≈ 94 is the starting load on day 1. One positive number is the whole alarm, the viral load is climbing, and climbing faster every day.
💡 What this means
a ≈ 94 is the starting load; b ≈ 0.303 the daily growth rate. On the log scale (right) the curve becomes a straight line, the signature of exponential growth.
⌨️ Stata command 
nlcom (doubling_time: ln(2) / _b[/b])
📋 Output, real least-squares fit (computed in Python)
   doubling_time |  Coef. = 2.29   [95% conf. int.: 2.26, 2.32]
📖 Read the output
Mirror of the half-life output. One number, 2.3 days, with a CI , the time for the load to double. This is the line a clinician acts on at the bedside, not the raw rate b above it.
💡 What this means
The mirror image of half-life. Doubling time = ln(2)/b ≈ 2.3 days in this within-patient curve, the clinically meaningful translation of the raw rate.
🦠 Outbreak lens
A growth rate of 0.3 is abstract; “doubles every couple of days” is what a clinician actually feels at the bedside.
⌨️ Stata command 
* ---- Epidemic scale: Gulu 2000, with a 30-day forward forecast ----
clear
set seed 43210
set obs 61
gen day = _n - 31                      // -30 (history) to +30 (forecast)
scalar b_rate = ln(2)/15.6             // 15.6-day doubling baked in

* True trend, then split into observed history and forecast
gen true_trend = 900*exp(b_rate*day)
gen history_cases  = round(true_trend + rnormal(0,20)) if day <= 0
replace history_cases = 0 if history_cases < 0
gen forecast_cases = round(true_trend + rnormal(0,50)) if day >= 0

* History plus the forward forecast on the unchecked exponential path
twoway (scatter history_cases day, mcolor(navy) msize(medium)) ///
       (line true_trend day if day <= 0, lcolor(navy) lwidth(medthick)) ///
       (scatter forecast_cases day, mcolor(cranberry) msize(small) msymbol(triangle)) ///
       (line true_trend day if day >= 0, lcolor(cranberry) lwidth(thick) lpattern(dash)), ///
    title("Ebola Case Trajectory & 30-Day Forward Forecast") ///
    subtitle("Unchecked Outbreak | Doubling Time: 15.6 Days") ///
    xtitle("Days (relative to today, Day 0)") ///
    ytitle("Cumulative confirmed/suspected cases") ///
    xline(0, lcolor(gs10) lpattern(dot)) xlabel(-30(10)30) ylabel(0(500)3500) ///
    legend(order(1 "Observed history" 2 "Historical trend" ///
                 3 "Projected forecast" 4 "Unchecked exponential path"))

* Recover the growth rate (shift time so the search starts clean)
gen combined_cases = history_cases
replace combined_cases = forecast_cases if day > 0
gen day_shifted = day + 30
nl (combined_cases = {a=23}*exp({b=0.04}*day_shifted))
nlcom (doubling_time: ln(2) / _b[/b])
📋 Output, real least-squares fit (computed in Python)
combined_cases    Coef.  Std.err.       t  P>|t|  [95% conf.  interval]
--------------  -------  --------  ------  -----  ----------  ---------
/a              235.390     4.042   58.23  0.000     227.302    243.479
/b               0.0446  0.000339  131.61  0.000      0.0439     0.0453

   doubling_time |  Coef. = 15.53   [95% conf. int.: 15.30, 15.77]
📈 Graph, generated from the data
📖 Read the output
Same two-parameter exponential, now fit to the whole district. We shift time (day_shifted = day + 30) so the search starts at a clean baseline instead of negative days, which helps it converge. The fitted b ≈ 0.045/day feeds straight into the doubling time of ≈ 15.5 days beneath it. Read the two numbers and ignore the shift: it moves a (the baseline count), never b (the growth rate).
💡 What this means
The same model at the population scale for the Gulu, Uganda outbreak (Sudan ebolavirus, 2000), with 30 days of history projected 30 days forward. The fitted rate b ≈ 0.045/day gives an epidemic doubling time of ≈ 15.5 days. The red dashed path is that same curve run forward, and nothing bends it down on its own. Contrast the within-patient doubling of about 2 days: same maths, two scales.
🦠 Outbreak lens
This early, near-invisible exponential phase is where outbreaks are won or lost. With cases doubling about every 15.5 days, every week you wait is another doubling, so the count you act on tomorrow is twice the one you ignored last week.
⚠️ Watch out
This is a reconstructed, illustrative trajectory for teaching the curve shape and the forecast logic, not an exact case record of the outbreak. A real forecast would also carry an uncertainty band.

Linear vs non-linear: which wins?

Fit both to the same auto data and let the numbers decide.

⌨️ Stata command 
regress mpg weight
estimates store linear_model
nl (mpg = {a}*exp({b}*weight)), initial(a 40 b -0.0005)
estimates store nl_model
estimates stats linear_model nl_model
📋 Output, real least-squares fit (computed in Python)
Model          AIC  Residual SD
----------  ------  -----------
Linear      1171.3         4.34
Non-linear  1142.1         4.19
📈 Graph, generated from the data
📖 Read the output
Two models, one scorecard, so read down the columns, not across. AIC: lower wins, and the non-linear row (1142.1) beats the linear one (1171.3). Residual SD says the same in plain units, the curve's typical miss (4.19) is smaller than the line's (4.34). Neither row means anything alone; the comparison between them is the whole point.
💡 What this means
AIC rewards fit and punishes complexity, lower is better. The non-linear model wins here (AIC 1142.1 vs 1171.3; residual SD 4.19 vs 4.34), because the straight line can't follow the bend. The gain is modest, so interpretability matters too.
⚠️ Watch out
Only compare AIC for models on the same outcome and rows.

Diagnostic checklist: run after EVERY fit

Six checks. The first is non-negotiable.

⌨️ Stata command 
di e(converged)
📋 Output, real least-squares fit (computed in Python)
1
📖 Read the output
The most important one-character output of the whole session. 1 means the search settled on a stable answer; a 0 would void every coefficient you read above it. So read this line first, before any estimate.
💡 What this means
e(converged) is 1 if the search succeeded, 0 if it gave up. If this isn't 1, stop, every other number is unreliable.
⌨️ Stata command 
rvfplot, yline(0)
qnorm resid
swilk resid       // p>0.05 = normal
predict cooksd, cooksd
summarize cooksd, detail
nl (...), initial(...) level(95)   // profile CIs for small n
💡 What this means
Then: residuals-vs-fitted (no pattern), Q-Q + Shapiro–Wilk (here a big p-value is good, p>0.05 means normal enough), Cook's distance for influence, and finally the human check: do the estimates make clinical sense?

Common Stata errors, and the fix

Non-linear fits fail in a handful of predictable ways.

MessageCauseFix
“could not calculate numerical derivatives”Poor starting valuesAdjust initial(); use trace to debug
“region of estimates is not feasible”Constraints violatedBound parameters with constraints()
“convergence not achieved”Iteration limit reachedRaise iterate(#) or improve initials
“singular matrix”Model over-parameterisedSimplify the model or get more data
“variance matrix is nonsymmetric”Hessian issuesTry vce(robust) or another technique()

When it won't converge

The most common failure, and the cures.

⌨️ Stata command 
nl (y={a}*exp({b}*x)), initial(a 40 b -0.0005) trace
nl (...), initial(...) iterate(100)
nl (...), initial(...) technique(gn)
💡 What this means
trace prints the residual sum-of-squares each step, smoothly shrinking is healthy, bouncing means bad starting values. iterate() and technique(gn) sometimes help, but nine times in ten the cure is a better starting guess read off the plot.

Hands-on exercise: infection rate vs bed occupancy

🅿️
🧩 Everyday version
Picture a car park filling up. While it's half-empty, finding a space is easy and stress stays low. Past a tipping point, say 75% full, it suddenly gets hard, tempers fray, and it climbs steeply toward gridlock. A hospital ward behaves the same as it fills: infections stay flat and safe for a while, then rise sharply once occupancy crosses a threshold. That's the logistic S-curve again, and your job is to find that threshold.

You're studying hospital bed occupancy (%) and infection rate (per 1000 patient-days). Infections stay low until occupancy crosses a threshold, then rise sharply to a ceiling , a logistic S-curve. Build it, fit it, interpret it.

⌨️ Stata command 
clear
set seed 999
set obs 30
gen occupancy = 40 + (_n-1)*(95-40)/29
* rate RISES with occupancy (note the minus sign in the exponent)
gen infection_rate = 12/(1+exp(-0.3*(occupancy-75))) + rnormal(0,0.5)
replace infection_rate = round(infection_rate, 0.1)
twoway scatter infection_rate occupancy
📈 Graph, generated from the data
💡 What this means
30 hospitals, occupancy 40→95%. True logistic: ceiling 12 infections / 1000 patient-days, threshold (inflection) at 75%, growth rate 0.3.
⚠️ Watch out
The deck's original formula used exp(0.3*(occupancy-75)), which makes the rate fall as wards fill, the opposite of the story. The minus sign fixes it.
⌨️ Stata command 
nl (infection_rate = {asym}/(1+exp(-{rate_g}*(occupancy-{inflect})))), ///
    initial(asym 12 rate_g 0.3 inflect 75)
nlcom (occupancy_50pct: _b[/inflect])
predict fitted_rate, yhat
predict resid_ex, resid
qnorm resid_ex
rvfplot, yline(0)
📋 Output, real least-squares fit (computed in Python)
infection_rate   Coef.  Std.err.       t  P>|t|  [95% conf.  interval]
--------------  ------  --------  ------  -----  ----------  ---------
/asym           11.793    0.2319   50.85  0.000      11.317     12.269
/rate_g         0.2956    0.0247   11.98  0.000      0.2450     0.3462
/inflect        74.445    0.3361  221.50  0.000      73.756     75.135
📖 Read the output
Now you read one unaided. asym ≈ 11.8 = the ceiling infection rate; inflect ≈ 74.4% = the occupancy at half-max, and for a logistic that inflection point is the danger threshold; rate_g ≈ 0.30 = the steepness. The one number that runs the ward is inflect: report it with its CI and you've answered the question.
💡 What this means
Really fitted: asym ≈ 11.8 = the ceiling rate; inflect ≈ 74.4% = the occupancy at half-maximum, and for a logistic the inflection point is the EC50; rate_g ≈ 0.30 = steepness.
⌨️ Stata command

Summary: the seven things to remember

#Takeaway
1Non-linear models give clinically meaningful parameters (ceiling, rate, EC50, half-life).
2Always plot first, the shape tells you which model.
3Starting values matter, guess from the plot, then refine.
4Check convergence, e(converged) must be 1.
5Diagnose residuals, rvfplot, qnorm, Cook's D.
6Report parameters with interpretation, not just numbers.
7Four go-to shapes: decay, growth, Michaelis-Menten, logistic.

Output glossary: every term, plain English

CoefficientThe estimated value of a parameter (a, b, vmax …). In non-linear models each has a real-world meaning, a ceiling, a rate, a midpoint.
Std. err.How much the estimate would wobble across repeated samples. Smaller = more precise.
P>|t|Probability of the estimate if the true value were zero. <0.05 = significant. (Exception: in normality tests like Shapiro–Wilk you WANT p>0.05.)
[95% conf. int.]Plausible range for the true value. Narrow = precise; excludes zero = significant.
yhat / fittedThe model's predicted value for each row, points on the fitted curve.
resid (residual)Actual − predicted. Good residuals are small, patternless and bell-shaped.
e(converged)1 = the search found a stable answer; 0 = it failed. Must be 1 first.
AICModel-comparison score that rewards fit and punishes complexity. Lower is better; compare only on the same data.
Cook's distanceHow much one data point moves the whole fit. Above 4/n = influential.
nlcomComputes a NEW quantity from fitted parameters (half-life, doubling time, EC50) with a correct CI.
Half-life / doubling timeln(2)/rate, the change that halves (decay) or doubles (growth) the outcome. The clinically intuitive translation of the rate.
EC50The dose (or level) giving half the maximum effect. In Michaelis–Menten it's Km; in a logistic it's the inflection point.
initial()Your starting guesses. The biggest cause of, and cure for, convergence problems.
