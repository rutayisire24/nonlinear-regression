Before you start — the 4-step rhythm

Every non-linear fit in this lecture follows the same four beats. If you can narrate these four, you can narrate the whole deck.

🖥️ Command
* 1. LOOK   plot the raw data first
* 2. GUESS  read starting values off the plot
* 3. FIT    nl (y = ...), initial(...)
* 4. CHECK  did it converge? are residuals clean?
💡 What this means
Linear regression lets you skip straight to the fit. Non-linear regression does not: the computer searches for the parameters by trial-and-improvement, starting from your initial guess. A bad guess means it may never find the answer. So we always plot first, guess sensibly, fit, then check.
🗣️ Say this
Non-linear models aren't solved with a formula — they're found by searching. Our job is to give the search a good place to start and to check where it ended up.

Model 1 — Exponential decay

Dataset: Stata's built-in auto. Clinical analogy: a drug concentration falling as body mass rises — fast at first, then flattening.

🖥️ Command
sysuse auto, clear
describe
list mpg weight in 1/5
📋 Representative output
Contains data from ...\auto.dta
 Observations:           74          1978 automobile data
    Variables:           12
------------------------------------------------------------
     +---------------------+
     | mpg   weight        |
  1. |  22    2,930        |
  2. |  17    3,350        |
  3. |  22    2,640        |
  4. |  20    3,250        |
  5. |  15    4,080        |
     +---------------------+
💡 What this means
sysuse auto, clear loads the demo dataset (the clear discards anything already in memory). describe lists the variables and how many rows. list … in 1/5 prints the first 5 rows so you can eyeball the numbers. Here mpg is the outcome (stand-in for drug concentration) and weight the predictor (body mass).
🗣️ Say this
We borrow the cars dataset as a pharmacokinetics stand-in: mileage behaves like drug concentration, weight like body mass.
🖥️ Command
twoway scatter mpg weight, ///
    title("MPG vs Vehicle Weight") ///
    xtitle("Weight (lbs)") ytitle("Miles per Gallon")
📈 Graph
A scatter of points sloping down and curving — high MPG for light cars, dropping steeply then levelling off for heavy ones.
💡 What this means
Always plot before fitting. The curved, decaying shape is the evidence that a straight line is wrong and an exponential decay curve is right. The /// just continues one long command onto the next line.
🗣️ Say this
See the curve? A straight line would under-predict the light cars and over-predict the heavy ones. That curve is why we go non-linear.
🖥️ Command
nl (mpg = {a} * exp({b} * weight)), ///
    initial(a 40 b -0.0005)
📋 Representative output
      mpg | Coefficient  Std. err.    t    P>|t|   [95% conf. int.]
----------+------------------------------------------------------
       /a |   41.23456     2.34500  17.58  0.000   36.567   45.902
       /b |   -0.000478    0.00007  -6.83  0.000  -0.00062 -0.00034
💡 What this means
nl fits a non-linear model. The {a} and {b} in curly braces are the unknowns Stata estimates. initial(a 40 b -0.0005) is your starting guess, read off the plot (ceiling ≈ 40 MPG, gentle decay). Reading the table: /a ≈ 41 is the ceiling (MPG at zero weight); /b ≈ −0.00048 is the decay rate — negative, so MPG falls as weight rises. Both P>|t| = 0.000, so both are clearly real, not chance.
🗣️ Say this
a is the starting height, b is how fast it falls. b is negative — that's decay. Tiny p-values mean we're confident both numbers are real.
⚠️ Watch out
If Stata prints “convergence not achieved” the numbers are meaningless — fix your initial() guesses and refit.
🖥️ Command
nl (mpg = {a} * exp({b} * weight)), ///
    initial(a 40 b -0.0005) vce(robust)
💡 What this means
Same model, but vce(robust) swaps in robust standard errors. The coefficients (/a, /b) don't change — only the Std. err., t and CI columns do. Use it when the residual spread isn't constant (common with decay/growth data). The conclusions usually hold; the uncertainty is just reported more honestly.
🗣️ Say this
Robust standard errors don't move the estimates — they make the error bars trustworthy when the scatter isn't even.
🖥️ Command
predict fitted_exp, yhat
predict resid_exp, resid
rvfplot, yline(0)
qnorm resid_exp
histogram resid_exp, normal
📈 Graph
(1) residuals vs fitted — should be a shapeless cloud around the zero line; (2) Q-Q plot — points should hug the diagonal; (3) histogram — should look roughly bell-shaped.
💡 What this means
predict … yhat saves the curve's predictions; predict … resid saves the residuals (actual − predicted). The three plots are the standard health-check: no pattern in residuals-vs-fitted, points on the line in the Q-Q plot, and a bell shape in the histogram all say the model fits well.
🗣️ Say this
Residuals are what the model couldn't explain. If they're patternless and bell-shaped, the model has squeezed out all the signal.
⚠️ Watch out
A funnel or U-shape in residuals-vs-fitted means the model shape is wrong — not just noisy.
🖥️ Command
nlcom (half_life: ln(2) / -_b[/b])
📋 Representative output
  half_life |   1450.2     177.4    8.17  0.000   1102.5   1797.9
💡 What this means
nlcom computes a new quantity from the fitted parameters and gives it a correct confidence interval. Here half-life = ln(2) / −b ≈ 1450 lbs: every extra ~1450 lbs of weight halves the MPG — exactly like a drug's half-life. _b[/b] means “the estimated value of parameter b.”
🗣️ Say this
This is the clinically intuitive number: how much change halves the outcome. Same maths as drug half-life.

Model 2 — Logistic growth (S-curve)

Dataset: Stata's uslifeexp (life expectancy 1900–2000). Clinical analogy: tumour growth, CD4 recovery, cognitive decline — slow, then fast, then a ceiling.

🖥️ Command
sysuse uslifeexp, clear
list year le in 1/10
📋 Representative output
     +-------------+
     | year   le   |
  1. | 1900  47.3  |
  2. | 1901  49.1  |
  3. | 1902  51.5  |
      ...
 10. | 1909  52.1  |
💡 What this means
New dataset. year is the predictor, le (life expectancy in years) is the outcome. We just load and peek at it. The values rise over time in an S-shape.
🗣️ Say this
Life expectancy over the 20th century: it climbs, accelerates, then levels off — the classic S-curve.
🖥️ Command
twoway scatter le year, ///
    title("US Life Expectancy Over Time")
📈 Graph
An S-shaped rising cloud: slow gains early, steep rise mid-century, flattening near the end.
💡 What this means
The S-shape (slow → fast → plateau) is the fingerprint of logistic growth. That's what tells us to fit a logistic curve rather than a line or a plain exponential.
🗣️ Say this
Three phases — lag, surge, ceiling. Whenever you see that, think logistic.
🖥️ Command
nl (le = {asym} / (1 + exp({rate} * (year - {inflect})))), ///
    initial(asym 80 rate -0.1 inflect 1950)
estimates store logistic_model
📋 Representative output
       le | Coefficient  Std. err.    t    P>|t|   [95% conf. int.]
----------+------------------------------------------------------
    /asym |   79.234      1.234    64.21  0.000   76.745   81.723
    /rate |   -0.095      0.011    -8.64  0.000   -0.117   -0.073
 /inflect | 1958.45       2.101   931.78  0.000 1954.222 1962.678
💡 What this means
Three meaningful parameters: /asym ≈ 79 is the ceiling (max life expectancy the curve approaches); /inflect ≈ 1958 is the year of fastest growth (the middle of the S); /rate sets the steepness. Tiny p-values + tight CIs = all three precisely estimated. estimates store saves the fit so we can compare models later.
🗣️ Say this
Asym is the ceiling, inflect is the tipping-point year, rate is how steep the climb is. All three are sharply estimated.
🖥️ Command
predict fitted_log, yhat
local asym = _b[/asym]
local rate = _b[/rate]
local inflect = _b[/inflect]
display "Predicted LE 2020: " `asym'/(1+exp(`rate'*(2020-`inflect')))
📋 Representative output
Predicted LE 2020: 79.06
💡 What this means
We pull each estimated parameter into a local (a named value) and plug them into the logistic formula for year 2020 to forecast. Result ≈ 79 years — close to the ceiling, because by 2020 the curve has nearly flattened.
🗣️ Say this
Once we have the parameters we can read the curve at any year — here, a 2020 forecast of about 79 years.
⚠️ Watch out
Forecasting far past your data is risky: the model assumes the same S-curve keeps holding.

Model 3 — Michaelis–Menten (saturating dose–response)

Simulated dose–response data (we set the true answer, so we can check the fit). Clinical analogy: receptor binding — response rises with dose then saturates.

🖥️ Command
clear
set seed 12345
set obs 40
gen dose = (_n - 1) * (100 / 39)
gen response = (90 * dose) / (25 + dose) + rnormal(0, 4)
💡 What this means
We simulate a study so we know the truth. set seed makes it reproducible; set obs 40 creates 40 rows; dose runs 0→100 mg/kg. The true curve has Vmax = 90 (max response) and Km = 25 (the EC50), plus random noise rnormal(0,4). The fit below should recover ~90 and ~25.
🗣️ Say this
We built this data ourselves with Vmax 90 and EC50 25 — so we can grade the model on whether it finds those numbers back.
🖥️ Command
nl (response = {vmax} * dose / ({km} + dose)), ///
    initial(vmax 90 km 25)
📋 Representative output
 response | Coefficient  Std. err.    t    P>|t|   [95% conf. int.]
----------+------------------------------------------------------
    /vmax |   89.456      2.345    38.15  0.000   84.712   94.200
      /km |   24.876      2.123    11.72  0.000   20.578   29.174
💡 What this means
/vmax ≈ 89 is the saturating maximum response. /km ≈ 25 is the EC50 — the dose that produces half of Vmax. Both land right on the true values (90 and 25), so the model works. In pharmacology Km/EC50 is the key potency number.
🗣️ Say this
Vmax is the most the drug can do; Km is the dose that gets you halfway there — the potency. We recovered both almost exactly.
🖥️ Command
predict cooksd_mm, cooksd
scatter cooksd_mm _n, yline(4/40)
📈 Graph
A spike plot of Cook's distance per observation, with a horizontal cut-off line at 4/40 = 0.1. Bars above the line are influential points.
💡 What this means
Cook's distance measures how much one single data point pulls the whole fit. The common cut-off is 4/n (here 4/40 = 0.1). Points above the line deserve a second look — a data-entry error, or a genuinely unusual patient.
🗣️ Say this
Cook's distance asks: would the curve move a lot if I deleted this one point? Anything above the line, we investigate.

Model 4 — Exponential growth (viral load)

Simulated viral load climbing after infection. Clinical analogy: early viral replication or untreated tumour growth.

🖥️ Command
clear
set seed 67890
set obs 20
gen day = _n
gen viral_load = 100 * exp(0.3 * day) + rnormal(0, 200)
💡 What this means
Same simulate-from-known-truth recipe. True start a = 100, true growth rate b = 0.3 per day. Note +exp(+0.3·day) — a positive exponent means growth (Model 1 had a negative one for decay).
🗣️ Say this
Same exponential family as Model 1, but the exponent is positive now — so instead of decay we get explosive growth.
🖥️ Command
nl (viral_load = {a} * exp({b} * day)), ///
    initial(a 100 b 0.3)
📋 Representative output
viral_load| Coefficient  Std. err.    t    P>|t|   [95% conf. int.]
----------+------------------------------------------------------
       /a |   98.456     12.34      7.98  0.000   73.456  123.456
       /b |   0.312       0.045     6.93  0.000    0.218    0.406
💡 What this means
/a ≈ 98 recovers the starting load (~100); /b ≈ 0.31 the daily growth rate (~0.3). Both close to the truth.
🗣️ Say this
Starting load about 100, growing ~31% a day — right where we built it.
🖥️ Command
nlcom (doubling_time: ln(2) / _b[/b])
📋 Representative output
 doubling_~e |   2.221     0.320    6.94  0.000    1.594    2.848
💡 What this means
The mirror image of half-life. Doubling time = ln(2) / b ≈ 2.2 days: the viral load doubles roughly every 2 days. This is the clinically meaningful translation of the raw rate b.
🗣️ Say this
A growth rate of 0.31 is abstract; “doubles every two days” is what a clinician actually feels.

Linear vs non-linear — which wins?

Fit both to the same data and let the numbers decide.

🖥️ Command
regress mpg weight
estimates store linear_model
nl (mpg = {a} * exp({b} * weight)), initial(a 40 b -0.0005)
estimates store nl_model
estimates stats linear_model nl_model
📋 Representative output
    Model |     N    ll(null)  ll(model)   df      AIC       BIC
----------+--------------------------------------------------
  linear  |    74      .        -195.0      2     394.0     398.6
  nl      |    74      .        -189.7      2     383.4     388.0
💡 What this means
AIC and BIC score model fit while penalising complexity — lower is better. Here the non-linear model has lower AIC and BIC (383/388 vs 394/399), so it fits the curved data better. The gap is modest, so interpretability also matters in the choice.
🗣️ Say this
Lower AIC/BIC wins. The exponential curve beats the straight line here — and it's also the more interpretable story.
⚠️ Watch out
Only compare AIC/BIC for models fit to the same outcome on the same rows. Don't compare across different datasets.

Diagnostic checklist — run after EVERY fit

Six quick checks. The first one is non-negotiable.

🖥️ Command
di e(converged)
📋 Representative output
1
💡 What this means
e(converged) is 1 if the search succeeded, 0 if it gave up. If this isn't 1, stop — every other number is unreliable. This is the single most important line in the whole lecture.
🗣️ Say this
First question, always: did it converge? One means yes. Zero means throw the results away and fix your starting values.
🖥️ Command
swilk resid
📋 Representative output
    Variable |   Obs    W      V      z     Prob>z
  -----------+----------------------------------
       resid |    74  0.981  1.23  0.46  0.3210
💡 What this means
Shapiro–Wilk formally tests whether residuals are normal. The rule: Prob>z (the p-value) > 0.05 ⇒ normal enough. Here 0.32 > 0.05, so residuals pass. (Counter-intuitively, here a big p-value is the good result.)
🗣️ Say this
Shapiro–Wilk: above 0.05 means the residuals are acceptably normal. Big p-value is good news this time.
🖥️ Command
predict cooksd, cooksd
summarize cooksd, detail
💡 What this means
Re-checks for influential points (see Model 3). summarize … detail shows the spread; watch the max — anything well above 4/n is worth investigating.
🗣️ Say this
One more pass for points that punch above their weight.
🖥️ Command
nl (y = {a} * exp({b} * x)), initial(a 10 b -0.1) level(95)
💡 What this means
level(95) requests 95% confidence intervals. For small samples the profile-likelihood CIs Stata can produce are more honest than the default symmetric ones. Also sanity-check: do the estimates make clinical sense?
🗣️ Say this
Last check is human, not statistical: do these numbers make sense for a real patient?

When it won't converge

The most common failure in non-linear regression — and the fixes.

🖥️ Command
nl (y = {a} * exp({b} * x)), initial(a 40 b -0.0005) trace
📋 Representative output
Iteration 0:  residual SS = 1.23e+04
Iteration 1:  residual SS = 8.91e+03
Iteration 2:  residual SS = 8.40e+03
...
💡 What this means
trace prints the residual sum-of-squares at each step of the search. If it keeps falling and settles, good. If it bounces or explodes, your starting values are sending the search the wrong way.
🗣️ Say this
Trace lets you watch the search think. Smoothly shrinking is healthy; bouncing means bad starting values.
🖥️ Command
nl (...), initial(...) iterate(100)
nl (...), initial(...) technique(gn)
💡 What this means
iterate(100) gives the search more attempts; technique(gn) switches to the Gauss–Newton algorithm. But the real fix is almost always better initial() values — read them off the plot.
🗣️ Say this
More iterations and a different algorithm sometimes help — but nine times out of ten the cure is a smarter starting guess.

Hands-on exercise — hospital infection rate

A logistic S-curve: infection rate stays low until ~75% bed occupancy, then rises sharply. Learners build, fit, and interpret it.

🖥️ Command
clear
set seed 999
set obs 30
gen occupancy = 40 + (_n - 1) * (95 - 40) / 29
gen infection_rate = 12 / (1 + exp(0.3*(occupancy-75))) + rnormal(0, 0.5)
💡 What this means
Build 30 hospitals with occupancy 40→95%. True logistic parameters: ceiling 12 infections / 1000 patient-days, inflection at 75% occupancy, growth rate 0.3.
🗣️ Say this
We're manufacturing a realistic safety signal: infections that stay flat until the ward gets dangerously full.
🖥️ Command
nl (infection_rate = {asym} / ///
    (1 + exp({rate_g}*(occupancy-{inflect})))), ///
    initial(asym 12 rate_g 0.3 inflect 75)
📋 Representative output
infection_~e| Coefficient  Std. err.   t    P>|t|  [95% conf. int.]
------------+----------------------------------------------------
      /asym |   12.234     0.345    35.46  0.000   11.527   12.941
    /rate_g |    0.312     0.045     6.93  0.000    0.220    0.404
   /inflect |   75.123     0.876    85.76  0.000   73.345   76.901
💡 What this means
/asym ≈ 12.2 = the ceiling infection rate. /inflect ≈ 75.1% = the occupancy at half the maximum — and for a logistic curve the inflection point is the EC50. /rate_g ≈ 0.31 = steepness. All recovered from the true values.
🗣️ Say this
The model nails the danger threshold at ~75% occupancy — that's the number that changes how you run the ward.
🖥️ Command
di "Occupancy at 50% max rate: " _b[/inflect] " (95% CI: " ///
   _b[/inflect] - 1.96*_se[/inflect] ", " ///
   _b[/inflect] + 1.96*_se[/inflect] ")"
📋 Representative output
Occupancy at 50% max rate: 75.123 (95% CI: 73.406, 76.840)
💡 What this means
Builds the headline sentence by hand: take the estimate _b[/inflect] and its standard error _se[/inflect], and form a 95% CI as estimate ± 1.96 × SE. The actionable message: keep occupancy below ~75% and infection rates stay low.
🗣️ Say this
Here's the policy line: below about 75% occupancy you're safe; above it, infections take off.

Model decision guide — pick the curve from the shape

ModelFormulaShapeClinical example
Exponential decayy = a · e^(b·x), b < 0Falls fast then flattensDrug clearance, radioactive decay
Exponential growthy = a · e^(b·x), b > 0Rises ever fasterEarly viral load, untreated tumour
Michaelis–Menteny = Vmax·x / (Km + x)Rises then saturates at a ceilingReceptor binding, dose–response
Logistic (sigmoid)y = asym / (1 + e^(rate·(x−inflect)))S-curve: lag, surge, plateauCD4 recovery, tumour growth, adoption

Output glossary — every term, plain English

CoefficientThe estimated value of a parameter (e.g. /a, /vmax). For non-linear models each one has a real-world meaning — a ceiling, a rate, a midpoint.
Std. err.Standard error: how much the estimate would wobble across repeated samples. Smaller = more precise.
tThe coefficient divided by its standard error — how many standard errors the estimate sits away from zero. Bigger (in absolute value) = stronger evidence it's not zero.
P>|t|The p-value. Probability of seeing this estimate if the true value were zero. Below 0.05 = statistically significant. (Exception: in normality tests like Shapiro–Wilk you WANT p > 0.05.)
[95% conf. int.]95% confidence interval — the plausible range for the true value. Narrow = precise. If it excludes zero, the effect is significant.
yhat / fittedThe model's predicted value for each row — points on the fitted curve.
resid (residual)Actual minus predicted. What the model couldn't explain. Good residuals are small, patternless, and bell-shaped.
e(converged)1 = the search found a stable answer; 0 = it failed. Must be 1 before you trust anything else.
AIC / BICModel-comparison scores that reward fit and punish complexity. Lower is better. Only compare models on the same data.
Cook's distanceHow much one single data point moves the whole fit. Above 4/n = influential, worth investigating.
nlcomComputes a NEW quantity from fitted parameters (half-life, doubling time, EC50) with a correct confidence interval.
initial()Your starting guesses for the parameters. The single biggest cause of (and cure for) convergence problems.
